# Детальное описание работы веб-приложения RouteRus

RouteRus — это веб-приложение для создания статических маршрутов для российских сервисов. Оно позволяет пользователям генерировать конфигурационные файлы различных форматов для настройки роутеров таким образом, чтобы российские сервисы не проходили через VPN-соединение.

## Архитектура приложения

Приложение построено на основе архитектуры MVC (Model-View-Controller) с использованием веб-фреймворка Flask и работает следующим образом:

### Серверная часть (Backend)

1. **Основной модуль (app.py)**:
   - Инициализирует Flask-приложение
   - Определяет конечные точки API и маршруты
   - Управляет сессиями и обработкой запросов

2. **Модуль генерации маршрутов (routerus.py)**:
   - Реализует основную логику поиска IP-адресов для доменов
   - Использует сервис rapiddns.io для получения IP-адресов доменов
   - Оптимизирует IP-адреса, объединяя их в подсети
   - Управляет хранением и обработкой данных доменов

3. **Модуль форматов маршрутов (route_formats.py)**:
   - Содержит форматы для различных типов устройств (Windows, Linux, роутеры и т.д.)
   - Преобразует IP-адреса и подсети в специфичные для каждого устройства форматы

### Клиентская часть (Frontend)

1. **HTML-шаблон (index.html)**:
   - Основная страница приложения с интерактивными элементами
   - Использует Bootstrap для стилизации и адаптивного дизайна

2. **JavaScript (main.js)**:
   - Управляет клиентской логикой и взаимодействием с пользователем
   - Обрабатывает асинхронные запросы к API через Fetch API
   - Обновляет интерфейс на основе полученных данных

3. **CSS-стили (styles.css)**:
   - Определяет внешний вид и оформление элементов
   - Дополняет стили Bootstrap для создания уникального дизайна

## Процесс работы

### 1. Загрузка и управление сервисами

Приложение хранит список российских сервисов в текстовом файле, организованном по категориям (банки, госсервисы, соцсети и т.д.). Пользователь может:

- Просматривать существующие категории и сервисы
- Добавлять новые сервисы в существующие категории
- Создавать новые категории сервисов

Файл сервисов загружается при запуске приложения и хранится в структурированном формате.

### 2. Генерация маршрутов

Процесс генерации маршрутов включает следующие этапы:

1. **Инициализация и подготовка**:
   - Пользователь выбирает желаемые форматы маршрутов
   - Система создает временные директории для выходных файлов
   - Запускается процесс в отдельном потоке для избежания блокировки интерфейса

2. **Поиск и обработка IP-адресов**:
   - Для каждого домена из списка сервисов система запрашивает IP-адреса через rapiddns.io
   - В случае отсутствия адресов, пытается использовать стандартное DNS-разрешение
   - Группирует IP-адреса по категориям сервисов

3. **Оптимизация маршрутов**:
   - Анализирует полученные IP-адреса для каждой категории
   - Объединяет смежные IP-адреса в подсети для уменьшения количества маршрутов
   - Оптимизирует маршруты по предпочтительным маскам подсети (/16, /24, /32)

4. **Генерация файлов**:
   - Создает файлы в выбранных форматах для каждой категории и общий файл со всеми маршрутами
   - Сохраняет результаты в именованных файлах и каталогах
   - Архивирует результаты для удобства скачивания

5. **Уведомление пользователя**:
   - Отображает прогресс обработки в реальном времени
   - Показывает сообщения об успешном завершении или возникших ошибках
   - Предоставляет ссылки для скачивания сгенерированных файлов

### 3. API и взаимодействие

Приложение предоставляет следующие API-эндпоинты:

- `/api/services` - получение списка сервисов по категориям
- `/api/add_service` - добавление нового сервиса в категорию
- `/api/generate` - запуск процесса генерации маршрутов
- `/api/status` - получение статуса текущей обработки
- `/api/download/{format_id}` - скачивание сгенерированных файлов
- `/api/available_formats` - получение списка доступных форматов

## Поддерживаемые форматы маршрутов

Приложение поддерживает следующие форматы маршрутов:

1. **Windows** - `route ADD {IP} MASK {MASK} 0.0.0.0`
2. **Linux/Unix** - `ip route add {IP}/{CIDR} via %gateway% dev %interface%`
3. **MikroTik** - `/ip firewall address-list add list={LIST_NAME} address={IP}/{CIDR}`
4. **Keenetic** - `ip route {IP} {MASK} interface global`
5. **OpenVPN** - `push "route {IP} {MASK}"`
6. **WireGuard** - `{IP}/{CIDR}` для секции AllowedIPs
7. **Cisco IOS** - `ip route {IP} {WILDCARD_MASK} %gateway%`
8. **Huawei** - `ip route-static {IP} {MASK} %gateway%`
9. **D-Link** - `create iproute {IP} {MASK} %gateway% 1`
10. **OpenWrt** - команды uci для настройки маршрутов
11. **TP-Link** - инструкции для настройки через веб-интерфейс
12. **ASUS** - инструкции для настройки через веб-интерфейс
13. **CIDR** - простой формат `{IP}/{CIDR}` для использования в различных системах

## Обработка ошибок и надежность

Приложение включает следующие механизмы для обеспечения надежности:

1. **Обработка сетевых ошибок**:
   - Повторные попытки при временных сбоях сети
   - Экспоненциальная задержка между попытками
   - Логирование ошибок для диагностики

2. **Валидация входных данных**:
   - Проверка формата доменных имен
   - Валидация IP-адресов и масок подсети
   - Предотвращение дублирования сервисов

3. **Многопоточность**:
   - Длительные операции выполняются в отдельных потоках
   - Ограничение числа одновременных запросов к внешним сервисам
   - Отображение прогресса в реальном времени

## Системные требования и зависимости

Для работы приложения требуется:

- Python 3.6 или выше
- Flask для веб-сервера
- Requests и Beautiful Soup для парсинга данных
- Библиотеки для работы с IP-адресами и DNS
- Bootstrap на клиенте для оформления

# Инструкция по установке и запуску RouteRus

Эта инструкция поможет вам установить и запустить веб-приложение RouteRus даже если вы новичок в программировании.

## Шаг 1: Клонирование проекта с GitHub

1. **Установка Git** (если у вас его еще нет):
   - Для Windows: Скачайте и установите Git с [git-scm.com](https://git-scm.com/download/win)
   - Для Mac: Установите через Terminal: `brew install git` (требуется [Homebrew](https://brew.sh/))
   - Для Linux: `sudo apt-get install git` (Ubuntu/Debian) или `sudo yum install git` (CentOS/Fedora)

2. **Клонирование репозитория**:
   - Откройте Терминал (Mac/Linux) или Командную строку/PowerShell (Windows)
   - Перейдите в папку, где хотите разместить проект: `cd Документы` или `cd Documents`
   - Выполните команду:
     ```
     git clone https://github.com/username/routerus.git
     ```
     (замените `username` на реальное имя пользователя GitHub)
   - Перейдите в папку проекта: `cd routerus`

## Шаг 2: Установка Python

1. **Скачивание и установка Python**:
   - Для Windows: Скачайте установщик с [python.org](https://www.python.org/downloads/) (выберите версию 3.6 или выше)
   - Для Mac: Скачайте установщик с [python.org](https://www.python.org/downloads/) или `brew install python3`
   - Для Linux: `sudo apt-get install python3 python3-pip` (Ubuntu/Debian) или `sudo yum install python3 python3-pip` (CentOS/Fedora)

2. **Проверка установки Python**:
   - Откройте терминал или командную строку
   - Введите: `python --version` или `python3 --version`
   - Убедитесь, что версия 3.6 или выше

## Шаг 3: Создание виртуального окружения

Виртуальное окружение позволяет устанавливать пакеты Python изолированно для конкретного проекта:

1. **Для Windows**:
   ```
   python -m venv venv
   venv\Scripts\activate
   ```

2. **Для Mac/Linux**:
   ```
   python3 -m venv venv
   source venv/bin/activate
   ```

Вы увидите, что в начале командной строки появился префикс `(venv)`, это означает, что виртуальное окружение активировано.

## Шаг 4: Установка зависимостей

В активированном виртуальном окружении выполните:

```
pip install -r requirements.txt
```

Эта команда установит все необходимые библиотеки, перечисленные в файле requirements.txt.

## Шаг 5: Настройка проекта

1. **Создание необходимых директорий**:
   ```
   mkdir -p data routes/windows routes/linux routes/mikrotik routes/keenetic routes/openvpn routes/wireguard routes/cisco routes/huawei routes/dlink routes/openwrt routes/tplink routes/asus routes/cidr
   ```

2. **Создание файла со списком сервисов** (если он не существует):
   ```
   touch data/russian_services.txt
   ```

## Шаг 6: Запуск приложения

1. **Запустите веб-приложение**:
   ```
   python app.py
   ```

2. **Откройте в браузере адрес**:
   - Введите в адресной строке браузера: `http://localhost:5000`
   - Вы должны увидеть главную страницу приложения RouteRus

## Шаг 7: Использование приложения

1. **Добавление сервисов**:
   - Выберите существующую категорию или создайте новую
   - Введите доменное имя сервиса (например, `sberbank.ru`)
   - Нажмите кнопку "Добавить сервис"

2. **Генерация маршрутов**:
   - Выберите нужные форматы маршрутов (например, Windows, Keenetic)
   - Нажмите "Сгенерировать маршруты"
   - Дождитесь завершения обработки
   - Скачайте файлы с маршрутами для выбранных форматов

## Шаг 8: Использование сгенерированных маршрутов

1. **Для Windows**:
   - Распакуйте скачанный ZIP-архив
   - Запустите файл .bat от имени администратора
   - Маршруты будут добавлены в таблицу маршрутизации Windows

2. **Для роутеров**:
   - Откройте административный интерфейс вашего роутера
   - Перейдите в раздел настройки статических маршрутов
   - Добавьте маршруты согласно инструкциям в скачанном файле

## Дополнительные советы

- **Остановка приложения**: Нажмите Ctrl+C в терминале, где запущено приложение
- **Деактивация виртуального окружения**: Введите команду `deactivate`
- **Обновление проекта с GitHub**:
  ```
  git pull origin main
  ```
- **Если возникли проблемы**:
  - Проверьте файл routerus_web.log для информации об ошибках
  - Убедитесь, что все зависимости установлены корректно
  - Проверьте настройки брандмауэра, если не можете подключиться к приложению

## Запуск на постоянной основе (для продвинутых пользователей)

Для запуска приложения на постоянной основе (чтобы оно работало даже после закрытия терминала), можно использовать:

1. **Для Linux/Mac (с использованием nohup)**:
   ```
   nohup python app.py > output.log 2>&1 &
   ```

2. **Для Windows**:
   - Создайте файл `start_routerus.bat` со следующим содержимым:
   ```
   @echo off
   cd /d %~dp0
   call venv\Scripts\activate
   start python app.py
   ```
   - Добавьте этот файл в автозагрузку Windows

Теперь вы полностью настроили и запустили RouteRus! Приложение поможет вам создавать статические маршруты для российских сервисов в различных форматах.
